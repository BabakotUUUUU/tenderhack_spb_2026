# Демонстрационные сценарии — TenderHack SPB 2026

## Архитектура 4-го источника (Рунет)

4-й источник — это собственный async-краулер, который:
- Обходит реальные российские интернет-магазины (НЕ WB/Ozon/YM)
- Извлекает товары через schema.org JSON-LD, microdata, OpenGraph
- Хранит результаты в SQLite FTS5 индексе
- Ищет по BM25 в локальном индексе
- Дополняет живым краулингом при каждом запросе
- **Не использует внешних поисковых API**

## Порядок демонстрации жюри

### 1. Показать исправление опечатки
```
Ввести: нотбук
Ожидание: CorrectionBanner "Исправлено: ноутбук", was_corrected=true
```

### 2. Показать нормализацию шин
```
Ввести: 205/55r16
Ожидание: система добавляет "шины 205/55 R16", category=tires
```

### 3. Показать группировку источников
```
Ввести: ноутбук lenovo
Показать StatsBar с 4 вкладками: ЯМ / Ozon / WB / Рунет
```

### 4. Показать 4-й источник (Рунет)
```
Ввести: шины 205/55 R16
Открыть вкладку "Рунет" — показать карточки с доменами shop.ru
Объяснить: это наш crawler, не поисковик
```

### 5. Показать кэш
```
Повторить запрос "ноутбук"
В логах появится: cache HIT, ответ мгновенный
```

## Тестовые запросы

### Оргтехника
| Запрос | Ожидание |
|--------|----------|
| `ноутбук` | WB + Ozon + ЯМ + Рунет (regard.ru, e2e4.ru и др.) |
| `нотбук` | was_corrected=true, corrected="ноутбук" |
| `принтер hp` | все источники |
| `мфу` | синоним раскрыт |
| `монитар` | rapidfuzz → "монитор" |

### Шины
| Запрос | Ожидание |
|--------|----------|
| `шины 205/55 R16` | все 4 источника |
| `205/55r16` | нормализовано → "шины 205/55 R16" |
| `зимняя резина` | typo → "зимние шины" |
| `зимние шины pirelli` | Рунет: 4tochki.ru, discounter.ru и др. |

### Одежда
| Запрос | Ожидание |
|--------|----------|

## API проверки (curl)

```bash
# Health
curl http://localhost:8000/api/health

# Опечатка
curl "http://localhost:8000/api/search?q=нотбук&region=Москва" | python3 -m json.tool | grep -E "was_corrected|corrected_query"

# Нормализация шин
curl "http://localhost:8000/api/search?q=205%2F55r16&region=Москва" | python3 -m json.tool | grep -E "corrected_query|category"

# Suggest
curl "http://localhost:8000/api/search/suggest?q=ноут"

# Результаты Рунета
curl "http://localhost:8000/api/search?q=шины+R16&region=Москва" | python3 -m json.tool | python3 -c "import sys,json; d=json.load(sys.stdin); print([r['source'] for r in d.get('results',[])])"
```

## Что говорить жюри о 4-м источнике

4-й источник (Рунет) реализован без каких-либо внешних поисковых API.
Архитектура полностью собственная:
- Playwright загружает поисковые страницы seed-сайтов (JS-рендеринг)
- Crawler следует по ссылкам → обнаруживает новые товарные страницы
- Extractor извлекает данные через JSON-LD / microdata / heuristics
- SQLite FTS5 хранит индекс — поиск без обращений к интернету
- Нет Google API, нет Bing API, нет DDG API, нет SearXNG
